1. Create S3 Bucket (Data Lake)

Go to S3.
Click Create bucket.
Bucket name: pick a globally unique name, example explo-capstone-ayush-2026.
AWS Region: ap-south-1.
Object Ownership: ACLs disabled.
Block Public Access: keep all enabled.
Bucket Versioning: disable for now.
Default encryption: SSE-S3 enabled.
Click Create bucket.
Open your new bucket.
Click Create folder.
Create folder clickstream.
Create folder curated.
Create folder athena-results.
Create folder firehose-errors.
Outcome: bucket and folder prefixes are ready.

2. Create Kinesis Data Stream

Go to Kinesis.
In left panel, choose Data streams.
Click Create data stream.
Data stream name: clickstream.
Capacity mode: On-demand (recommended for MVP).
Click Create data stream.
Wait until Status becomes Active.
Outcome: stream clickstream is active.

3. Create Firehose Delivery Stream (Kinesis -> S3)

Go to Kinesis Data Firehose.
Click Create Firehose stream.
Source:
Source type: Amazon Kinesis Data Streams.
Kinesis data stream: select clickstream.
Destination:
Destination type: Amazon S3.
S3 bucket: select your bucket.
S3 bucket prefix: clickstream/.
S3 error output prefix: firehose-errors/.
Buffer hints:
Buffer size: 5 MiB.
Buffer interval: 60 seconds.
Compression: GZIP (recommended).
Encryption: Enabled.
Record transformation: Disabled for MVP.
Dynamic partitioning: Disabled for MVP.
CloudWatch logging: Enabled.
IAM role:
Choose Create new or update IAM role.
Role name can be auto-generated by wizard.
Click Create Firehose stream.
Wait until Firehose status is Active.
Outcome: Firehose is continuously delivering Kinesis records to S3 clickstream prefix.

4. Verify Firehose Delivery

Keep producer off for now, first verify health.
Open Firehose stream Monitoring tab.
Check metrics pane is visible.
Later you should see Incoming records and Delivery to S3 metrics.
Open S3 bucket clickstream folder.
Confirm new .gz objects appear after producer starts.
Outcome: delivery path validated.

5. Create Glue Database

Go to AWS Glue.
Left panel Data Catalog -> Databases.
Click Add database.
Name: explo_capstone.
Description: Inventory demand intelligence dataset.
Click Create database.
Outcome: Glue catalog database exists.

6. Create Athena Query Result Location

Go to Athena.
If first time, open Settings in the editor page.
Set Query result location to:
s3://your-bucket-name/athena-results/
Save settings.
Outcome: Athena can run queries without errors.

7. Create Athena Tables from SQL

In Athena editor, select Workgroup primary.
Ensure Data source is AwsDataCatalog.
Paste and run create database statement if needed:
CREATE DATABASE IF NOT EXISTS explo_capstone;
Set database dropdown to explo_capstone.
Create raw clickstream table (JSON in S3):
Run:
CREATE EXTERNAL TABLE IF NOT EXISTS explo_capstone.clickstream_raw (product_id string, event string, timestamp string, user_id string) ROW FORMAT SERDE 'org.openx.data.jsonserde.JsonSerDe' LOCATION 's3://your-bucket-name/clickstream/' TBLPROPERTIES ('has_encrypted_data'='false');
If you also want curated sales table:
Upload your local sales file to s3://your-bucket-name/curated/sales_data/ as sales_data.csv.
Then run curated table SQL from athena_glue_setup.sql.
Outcome: Athena tables are queryable.

8. Run Analytics Queries in Athena

In Athena editor, database explo_capstone selected.
Run forecast and surge SQL from athena_metrics.sql.
First test raw data:
SELECT * FROM clickstream_raw LIMIT 20;
Then run moving average and surge query.
Then run safety stock and reorder point query.
Save successful queries with names for demo.
Outcome: metrics are produced for dashboard use.

9. Start QuickSight (Dashboard Layer)

Go to QuickSight.
If first time, complete account setup.
In top-right, choose Manage QuickSight.
Security and permissions:
Allow access to Athena and S3 buckets used by Athena results and data.
Return to homepage and click New dataset.
Choose Athena.
Data source name: explo_capstone_athena.
Workgroup: primary.
Choose database explo_capstone.
Select table or view from metrics query outputs.
Click Visualize.
Build visuals from kpi_spec.md:
Demand trend line chart.
Forecast vs adjusted demand line chart.
Safety stock and ROP bar chart.
Surge alert table using conditional formatting.
Add filters for date and product.
Save analysis as dashboard.
Outcome: evaluator-ready dashboard.

10. Local Project Config Update

Open .env.example as reference.
Create a local .env file in project root.
Set values:
AWS_REGION=ap-south-1
KINESIS_STREAM_NAME=clickstream
EVENT_INTERVAL_SECONDS=1
S3_OUTPUT_BUCKET=your-bucket-name
Run producer:
python generate_sample_data.py for local data testing.
python clickstream_producer.py for live Kinesis events.
Outcome: local app points to your real AWS resources.

11. GUI Validation Checklist (Must Pass)

Kinesis stream status is Active.
Firehose stream status is Active.
Firehose Monitoring shows non-zero incoming records.
S3 clickstream folder is receiving new files.
Athena query SELECT * FROM clickstream_raw LIMIT 20 returns rows.
QuickSight visuals render and refresh.
12. Common GUI Mistakes to Avoid

Region mismatch between services.
Missing Athena result location.
Firehose source selected as Direct PUT instead of Kinesis Data Streams.
Wrong S3 prefix in table LOCATION.
QuickSight missing S3 permission for Athena output bucket.
Producer running with wrong stream name.
