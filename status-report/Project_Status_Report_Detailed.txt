1) PROJECT OVERVIEW
====================================================================
This project builds an end-to-end supply-chain analytics pipeline that starts from user behavior simulation (clickstream), stores data in AWS, processes it into business metrics, and prepares results for decision dashboards.

Main business problem solved:
- Retail teams need early warning of demand spikes and stockout risk.
- Raw events are converted into operational inventory metrics.

Primary objective:
- Predict demand trend and convert it into actionable inventory controls using:
  - Moving Average Forecast
  - Surge Index
  - Adjusted Demand
  - Safety Stock
  - Reorder Point (ROP)

MVP success criterion:
- "System runs end-to-end on sample data" (achieved).


====================================================================
2) FINAL ARCHITECTURE IMPLEMENTED
====================================================================
Pipeline flow implemented:
1. Local Python producer simulates clickstream events.
2. Events are pushed to Amazon Kinesis Data Stream.
3. Amazon Kinesis Firehose delivers stream data into Amazon S3.
4. AWS Athena queries S3 data using external tables.
5. SQL logic computes forecast and inventory metrics.
6. Outputs are ready for Amazon QuickSight dashboard consumption.

AWS resources connected:
- Region: ap-south-1
- Kinesis stream: clickstream
- Firehose stream: Capstone-Firehose-Stream
- S3 bucket: explo-capstone-ayush
- S3 prefixes used:
  - clickstream/
  - curated/sales_data/
  - athena-results/
  - firehose-errors/
- Athena DB: explo_capstone

7) MAJOR ISSUES FACED AND RESOLUTIONS
====================================================================
Issue 1: Athena column mismatch for clickstream fields (price/device not resolvable).
Resolution:
- Standardized clickstream table to stable fields present in current stored data.

Issue 2: CSV date parsing error in Athena (BAD_DATA with date type).
Resolution:
- Stored date column as string in external table and casted to date in analytical queries.

Issue 3: SQL context confusion in Athena (default DB vs target DB).
Resolution:
- Explicitly used explo_capstone DB and fully qualified table names.

Issue 4: Terminal visibility requirement.
Resolution:
- Commands executed in foreground so logs/output are directly visible.
